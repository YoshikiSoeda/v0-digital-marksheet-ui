### PR #101 — fix(students): id ベースの PATCH/DELETE で students-list の致命バグを潰す (PR #98 と同型)

**バグ A**: `students-list/page.tsx` の `handleSaveEdit` は saveStudents で students 全件を `register_student_canonical` (`ON CONFLICT (university_code, student_id)`) に POST。student_id を変えると新キーで既存行が一致せず、新規 INSERT で別 id 行が作られ → 旧 id 行は orphan として残存。

**バグ B**: `handleDelete` / `handleResetAllData` は saveStudents(filtered) を呼んでいたが、内部の upsert RPC は DB から行を取り除けない。**UI 上だけ消えて DB はそのまま** (silent fail)。PR #98 (patients-list) と同型。

**修正**:
- 新設 `PATCH /api/students/[id]` — id ベース UPDATE + Y-2 scope + UNIQUE (univ, student_id) 衝突 → 409
- 新設 `DELETE /api/students/[id]` — `student_test_session_assignments` は CASCADE で自動削除、attendance/exam_results は `student_id` (text) 参照 (FK ではない) なので残存 (過去評価履歴の保全)
- `lib/api/students.ts` `updateStudent(id, input)` / `deleteStudentApi(id)`
- `lib/data-storage.ts` `deleteStudent(id)` (deleteTeacher / deletePatient と対称)
- UI: handleSaveEdit を PATCH に切替、handleDelete / handleResetAllData を DELETE に切替

**動作確認**: 本番 DB で BEGIN..ROLLBACK にて UPDATE 経路 / DELETE → CASCADE を検証。orphan / orphan_assignments=0。
